# 软件测试智能体实验报告

## 一、实验目的
本实验目标是设计并实现一个“软件测试智能体（Software Testing Agent）”，使其能够自动或半自动完成测试用例生成、测试执行和结果分析，形成可运行的自动化测试闭环，并应用于真实软件对象。

## 二、被测软件项目
- 被测对象：本机安装的 `markdown2` 包本体
- 版本：`2.5.5`
- 源码路径：`/opt/homebrew/anaconda3/lib/python3.12/site-packages/markdown2.py`
- 主要功能：将 Markdown 文本转换为 HTML

说明：本项目不是仅测试本地 wrapper 逻辑，而是直接调用和分析本机 `markdown2` 包本体，满足“真实软件项目测试”的要求。

## 三、测试智能体设计与实现
系统采用模块化设计，核心模块如下：
1. `llm_agent.py`：调用 LLM（DeepSeek）进行源码分析、测试用例生成与失败分析。
2. `code_reader.py`：读取被测源码（动态定位到本机 markdown2 源码）。
3. `test_generator.py`：生成测试用例（LLM 优先，失败时兜底规则用例）。
4. `test_runner.py`：执行测试（单元层 + 集成层）。
5. `result_analyzer.py`：判定 PASS/FAIL/CRASH，并进行跨层一致性判定。
6. `whitebox_analyzer.py`：统计白盒覆盖率（行覆盖率、分支覆盖率）。
7. `report_generator.py`：输出测试报告和失败分析。

## 四、如何满足实验要求（逐条对应）
1. 自动或半自动测试能力  
已满足。系统可自动完成：生成测试输入 -> 执行 -> 判定 -> 生成报告。

2. 测试方法可选、可组合（黑盒/白盒）  
已满足。当前实现为组合方案：
- 黑盒：基于 `must_contain` 的输出断言；
- 白盒：基于 `coverage.py` 的行/分支覆盖率统计。

3. 测试层级（单元/集成/系统）  
已满足核心要求，当前实现：
- 单元测试：函数级调用 `markdown2.markdown(...)`；
- 集成测试：子进程执行 `python -m markdown2`；
- 系统测试目前为可扩展项（已具备扩展基础）。

4. 自动生成测试用例并执行  
已满足。默认生成上限可配置（当前主流程配置为 50），并有兜底机制保证流程不因 LLM 波动中断。

5. 记录测试过程与结果、分析缺陷  
已满足。输出：
- `results/report.md`：测试统计、方法层级统计、覆盖率、失败分析；
- `results/failed_cases.json`：失败用例明细（含输入、输出、判定）。

6. 对缺陷场景给出分析和建议  
已满足。对失败样例可给出根因分类、影响范围、优先级、建议负责人和改进建议（LLM 或本地规则）。

## 五、测试流程
1. 读取被测对象源码并做静态分析（LLM）。
2. 生成测试用例（优先 LLM，失败则回退兜底用例）。
3. 对每个用例执行双层测试（单元层、集成层）。
4. 汇总判定并检查跨层输出一致性。
5. 统计白盒覆盖率（行/分支）。
6. 输出报告与失败分析结论。

## 六、实验效果与结论
本项目已实现一个可实际运行的软件测试智能体，具备自动化测试闭环能力，并在真实软件对象（本机 markdown2 包）上完成测试实践。系统不仅能进行黑盒验证，还能给出白盒覆盖率指标，并支持多层级执行与差异发现，整体满足本实验的主要要求。

## 七、当前不足与后续优化
1. 覆盖率仍有提升空间，可继续扩展更复杂 Markdown 场景。
2. 可进一步增加系统测试场景（文件批处理、异常恢复、性能压力）。
3. 可将断言从字符串片段升级到结构化 DOM 断言，提升判定精度。
